# Mini Orchestrator — Security

**Version**: 1.0  
**Last updated**: 2026-04-16

---

## 1. Threat Model

**Context**: local-only, single-user developer tool. No authentication, no multi-tenancy, no cloud exposure. The attack surface is narrow but not zero.

| Asset | Description |
|-------|-------------|
| Claude Max session | `claude` CLI auth; if compromised, attacker can burn subscription quota |
| Disk artifacts | Generated design docs; potentially contain sensitive product ideas |
| SQLite DB | Project metadata + chat history |
| Env config | `.env` file; may contain `GITHUB_TOKEN` for Stage 2 |

| Attacker | Vector |
|----------|--------|
| Local process / malicious dep | Reads `~/.claude/` credentials, `.env`, SQLite |
| Malicious agent output | Prompt injection via generated artifact content |
| Path traversal via API | Read arbitrary files via artifact endpoint |
| Accidental secret exfiltration | API key committed to git or exposed in subprocess env |

Trust boundary: everything runs on the developer's machine, behind localhost. No external traffic except outbound `claude` CLI calls.

---

## 2. API Key Protection

**Rule**: `ANTHROPIC_API_KEY` is stripped from the subprocess environment before every `claude -p` call.

```python
env = os.environ.copy()
env.pop("ANTHROPIC_API_KEY", None)
proc = await asyncio.create_subprocess_exec(*cmd, env=env, ...)
```

**Why**: If an API key is present, the CLI would use API billing instead of the Max subscription. Stripping it forces the Max auth path and prevents accidental charges. The app is designed to work with zero API keys.

## 3. Input Validation

### Artifact Path Safety
The artifact read endpoint enforces basename-only filenames:
```python
safe = os.path.basename(filename)
if safe != filename or not safe.endswith(".md"):
    raise HTTPException(status_code=400, detail="invalid filename")
```
This prevents path traversal (`../../etc/passwd`).

### API Inputs
- `instruction` for `/revise`: stripped + non-empty check
- `idea` for `/launch`: stripped + non-empty check
- `content` for `/chat` and `/notes`: stripped + non-empty check
- Role values in `affected_roles`: validated against `AgentRole` enum; `LEAD` and `REVIEWER` are explicitly rejected

### Agent Inputs
Agent user messages are constructed from user-supplied idea text and generated artifact content. These are passed as CLI arguments, not shell commands — no shell injection risk (`asyncio.create_subprocess_exec` takes an argv list, not a shell string).

## 4. Secrets Management

| Secret | Location | Handling |
|--------|----------|----------|
| Claude Max auth | `~/.claude/` (managed by CLI) | Never touched by this app |
| `GITHUB_TOKEN` | `.env` | Gitignored; not yet used (Stage 2) |
| `ANTHROPIC_API_KEY` | Should not exist | Stripped at subprocess boundary |

**Pre-commit hook blocks**:
- `sk-ant-*` patterns
- `ghp_*`, `gho_*`, `github_pat_*`
- AWS access key patterns
- PEM private key headers
- `.env` files (`.env.example` is allowed)

---

## 5. Data Protection

- **No PII by default**: the app processes project ideas and generates design docs. No user accounts, no email, no payment data.
- **Chat history**: stored in SQLite `lead_messages`. Contains whatever the user typed. Cleared by deleting `data/orchestrator.db`.
- **Generated artifacts**: stored under `backend/output/` (gitignored). May contain sensitive product ideas. User is responsible for managing this directory.
- **In-transit**: all traffic is `localhost` only. No TLS needed.
- **At-rest encryption**: not provided. Relies on OS-level disk encryption (FileVault, etc.).

---

## 6. Prompt Injection Risk

Agent prompts are constructed by concatenating:
- The system prompt (static, from `stage1.py`)
- The user's idea (user input)
- Prior artifact content (AI-generated by earlier agents)

A malicious earlier artifact could attempt to redirect a later agent (e.g., a PRD that instructs the Backend agent to ignore its system prompt). Mitigations:
- All Stage 1 agents have `--tools ""` — no tool execution, so injected instructions cannot take file or system actions
- The Reviewer re-checks artifacts before Stage 2; any anomalous content would surface
- Stage 2 coding agents (not yet built) will require careful prompt structure to isolate agent instructions from artifact content

This is an accepted risk for the current Stage 1 scope.

---

## 7. CORS

The FastAPI app allows `http://localhost:5173` only:
```python
allow_origins=["http://localhost:5173"]
```
Not a wildcard. A malicious page on a different origin cannot call the API.

---

## 8. Dependency & Supply Chain

- Python deps: `pyproject.toml` / `requirements.txt`. No automatic updates.
- Node deps: `package.json` with lock file (`package-lock.json`).
- Pre-commit hook: `ruff check` on Python, `tsc --noEmit` on TypeScript.
- No automated CVE scanning currently. Planned for Stage 2 when the attack surface grows.

---

## 9. Non-Goals

- Authentication / authorization (single-user local tool)
- Rate limiting (no external exposure)
- Audit logging (no compliance requirements)
- Network isolation of agent subprocesses
- Sandboxing of generated code (Stage 2 concern — not yet built)
